Detailed Description

Decouples getting the various contextes from the actual type of context, so that the DataAllocator does not need to know about the various serialization methods. Since there is only a few context types we support, this can be done in an ad hoc manner making sure each overload of ContextRegistry<T>::get() uses a different entry in ContextRegistry::contextes;

Right now we use:

MessageContext 0 ROOTObjectContext 1 StringContext 2 ArrowContext 3
